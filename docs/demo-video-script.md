# Demo Video Script

Target length: 60-90 seconds.

Goal: show Finance Tracker as a complete mobile app project preparing for Play Store release, with the AI assistant as the differentiator.

## Suggested Flow

### 0-8 seconds: Open the app

Show the app launching into the welcome/auth experience.

Voiceover:

"This is Finance Tracker, a mobile-first personal finance app built with Expo React Native, FastAPI, Supabase, and Gemini AI."

### 8-18 seconds: Log in

Sign in with a prepared test account. Keep this quick and avoid typing slowly on camera.

Voiceover:

"The app uses Supabase authentication, and each user's transactions and budgets are scoped to their own account."

### 18-32 seconds: Dashboard overview

Show the Home dashboard with balance, income, spending, quick-add shortcuts, AI insight, and budget preview.

Voiceover:

"The dashboard gives a quick snapshot of cash flow, recent activity, budget progress, and AI-supported warnings."

### 32-45 seconds: Add a transaction

Open Add, enter a small sample expense or income, save it, and show the success feedback.

Voiceover:

"Users can add transactions manually or through quick-add shortcuts, with offline queue support when connectivity is limited."

### 45-57 seconds: View budgets

Scroll to budgets or open the relevant budget area. Show progress bars and categories.

Voiceover:

"Budgets help users compare category spending against monthly limits and understand where they are close to overspending."

### 57-75 seconds: Use the AI assistant

Open the AI tab and ask a prepared question such as:

"What category should I cut back on first this week?"

Use a successful response if possible.

Voiceover:

"The AI assistant runs through the backend, using summarized finance context from Supabase. API keys stay server-side, and daily AI usage limits help control cost."

### 75-90 seconds: Show GitHub README

Briefly switch to the GitHub README and scroll through the architecture/security sections.

Voiceover:

"The project includes a Vercel-ready FastAPI backend, Supabase migrations with RLS policies, EAS Android build profiles, and a Play Store release path after final compliance setup."

## Screens To Avoid In This Video

- Full transaction history, because the dedicated history screen is planned.
- Receipt scan if permissions or AI latency make the recording unpredictable.
- Market search if results are inconsistent.
- Any screen showing backend cold-start errors or validation errors.
- Long settings/import/export flows unless you need a quick proof of completeness.

## Recording Notes

- Use a test account with sample data already prepared.
- Start the backend or deployed API before recording.
- Open the app once before recording to avoid cold-start delay in the final take.
- Keep the demo focused on the strongest path: login, dashboard, add transaction, budgets, AI, README.
